Lesson #1
Git Branching Strategy is Essential
Implemented develop ‚Üí staging ‚Üí main workflow. Never push directly to main! Branch-based deployments prevent production failures and enable parallel feature development.
DevOps Best Practice
Lesson #7
GitHub Actions OIDC > Access Keys
Migrated from long-lived AWS Access Keys to OIDC authentication. Zero credentials in GitHub Secrets, temporary tokens only, fully audit-logged.
Security IAM
Lesson #27
IAM Hybrid Approach Works Best
Manual IAM Role creation + Terraform for infrastructure. Prevents circular dependencies and state corruption. GitHubActionsRole created once, referenced in all modules.
Terraform Architecture
Lesson #33
Terraform Seed Module = 100% Reproducibility
Database seeding via Terraform null_resource ensures zero manual setup steps. Test user, products, categories automatically created on deploy. From zero to production in ~10 minutes.
Automation DevOps
Lesson #31
Incremental Deploys - The Game Changer
Switched from destroy-redeploy to incremental updates. Lambda-only deploys in 3 minutes vs 15 minutes full stack. Preserves state, reduces errors, faster iteration.
Performance Workflow
Lesson #2
Terraform State is CRITICAL
Learned the hard way: local state is fragile. Implemented remote state (S3 + DynamoDB locks) for team collaboration. Always run terraform plan before apply.
IaC State Management
Lesson #25
CloudWatch Monitoring with Terraform
9 CloudWatch Alarms deployed via Terraform: Lambda errors, API Gateway 5xx, DynamoDB throttling. SNS email notifications for proactive issue detection. Production-ready monitoring from day one.
