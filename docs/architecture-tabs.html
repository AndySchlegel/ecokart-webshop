GitHub Actions OIDC - Eliminating Static AWS Keys
Phase 1: Security Excellence
Old Approach
AWS access keys stored as GitHub secrets. Long-lived credentials with manual rotation required. Major security risk if secrets leak or repository is compromised.
Modern Approach
Implemented GitHub OIDC with AWS IAM roles. Short-lived tokens (15 min lifetime), no secrets stored in GitHub, automatic rotation. GitHub workflow authenticates directly with AWS using federation trust.
No Stored Keys
15 Min Token Lifetime
Auto Rotation

The Crisis
After changing deployment architecture from terraform/examples/basic/ to terraform/ root, state became corrupted. Error: "Provider configuration not present". Resources existed in AWS but Terraform couldn't map them. All infrastructure operations blocked.
The Solution

Amplify Webhook Permissions - 8 Iterations to Success
Phase 1: IAM Deep Dive
The Error
AccessDeniedException: amplify:CreateWebhook on resource. GitHub OIDC role couldn't create Amplify webhooks for automatic deployments. Tried 7 different IAM policy combinations, all failed.
The Solution
IAM policy needs TWO separate statements: CreateWebhook on APP resource (arn:aws:amplify:*:*:apps/*), AND GetWebhook on WEBHOOK resource (arn:aws:amplify:*:*:apps/*/webhooks/*). Different actions operate on different resource types!
8 Iterations
Granular IAM Permissions
Trial & Error Documented
5
Cost Optimization - Environment-Based Sizing
Phase 1: Business Value
The Insight
Development doesn't need production-level resources. Running 3 identical environments (dev, staging, prod) at full capacity would cost 360 EUR/month - unsustainable for learning project!
The Strategy
Dev: 256 MB Lambda + PAY_PER_REQUEST (~25 EUR). Staging: 512 MB + low provisioned (~50 EUR). Prod: 1024 MB + high provisioned (~120 EUR). Total: 195 EUR/month. Destroy dev when idle saves another 75%!
45% Cost Reduction
Right-Sized Resources
Destroy Dev When Idle

Frontend Token Storage Bug - The Invisible Problem
Phase 1: Technical Debugging
The Mystery
Login worked, console showed "User logged in", Lambda logs showed "JWT validated successfully", but cart requests returned 401 Unauthorized. localStorage was EMPTY! Token was received but never persisted.
The Fix
Frontend authentication code received tokens from Cognito but didn't save them to localStorage. Added explicit token persistence after login/registration. Now tokens survive page refreshes and subsequent requests work.
State Management Critical
12 Hours Debugging
Silent Failure Pattern

Stripe Webhook Handler - Gateway to Custom Domains
Phase 2: Real-World Integration
The Challenge
Async payment flow: Customer redirected to Stripe â†’ Payment â†’ Webhook callback. Webhook signature verification failed due to rotated secret mismatch. Cart not clearing because wrong DynamoDB key (cart.id vs userId). This was the catalyst for implementing complete custom domains!
The Implementation
Lambda webhook handler validates Stripe signature (synced secret!), extracts metadata (userId, shippingAddress), creates order in DynamoDB, deducts stock, clears cart (using correct userId key!), triggers email. Idempotent - handles duplicate webhooks safely.
